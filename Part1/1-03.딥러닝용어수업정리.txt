Model
특징을 뽑아내고 예측을 하는 것이 모델안에서 이뤄진다
CNN은 모델이다 모델은 CNN이다
어떤 모델을 썼느냐? 몇개로 사용할 것이냐?

Layer
딥러닝은 예전에는 단순하게 적게만 사용했다
하지만 이제는 층을 여러 층을 쌓아서 딥러닝이라 한다
Layer층을 쌓는게 딥러닝의 핵심이다
Input layer 입력
가운데 Hidden layer
Output layer 출력
어떻게 쌓느냐가 모델의 구조가 된다
VVG16은 대략적으로 모델 중 하나로 옥스포드에서 만들었다
몇 층, 몇개 를 쌓았냐에따라서 16은 대략적인 층수를 말하기도 한다
많이 쌓는다고 해서 좋은 것도 아니다
오버피팅이 되거나 성능적으로 무겁거나 느릴 수 있어서
단점도 있다
효율적으로 쌓는게 더 좋다
Layer층은 깊이 쌓을 수록 디테일하게 뽑는 장점도 있긴 하다

Convolution (특징을 뽑는 레이어층)
CNN에서 C가 Convolution이 합성법이라고 하는데
어떤 이미지가 있으면 필터를 받는다 필터를 받으면
결과를 만들어낸다 즉, 테두리 필터를 만든다

Weight/Filter/Kernel/Variable/Bias
서로 다른 말이긴 한데
Convolution안에서 이뤄지는 것들이다
Weight는 학습하려고 하는 대상이다
필터가 고정된 값이 아니고 바꿔가면서 더 잘 뽑게 하려고 한다
더 좋은 필터 더 좋은 특징을 찾는 것을 만들기 위해
더 좋은 계산을 하게 만드는 것이다
데이터나 목적에 따라 다른 Weight를 학습시켜야 된다

Pooling Layer는 (압축을 하는 층)
압축을 한다
약간 이미지가 32x32를 16x16, 14x14로 사이즈를 조금 다르게 줄일 수
있지만 이미지가 가지고 있는 가장 큰 특징 Max Pooling을 반으로 줄여준다

Optimization은
인공지능을 학습시키고 나서 최적화하는 것이다
Lost Function으로 나온 결과를 계산해서 모델에다 업데이트를
해줘서 오류를 최소한으로 줄여야 된다
줄일 수 있도록 도와주는 역할을 한다
Loss를 줄이기 위한 역할이 Optimization이다

Activation Function
앞에서 Convolution하고나서 특징을 뽑으면
음수 값이나 불필요한 값을 제거한다
ReLU가 대표적으로 많이 사용한다
불필요한 연산도 줄여준다

Softmax는
마지막에 최종에서 컴퓨터가 예측을 한다
수치에 따라 강한 곳에 정답을 유도한다
1.5 2 1 0.5 0.1일때 2가 정답이라고 한다
컴퓨터가 계산해서 확률을 주는데 
SOFTMAX가 앞에 받은 값들을 확률로 계산해서 출력한다

Cost/Loss/Loss Function는
결과로
0,1,2,3,4가 있는데
어떤 문제가 정답이 3번이 정답이라고 하면
얼마나 틀렷는지 계산을 해준다
각각 얼마나 틀렸는지 계산하는게
Lost Function인데 거기서 Cost와 Loss로 구별된다

Learning Rate
는 컴퓨터가 직접 설정은 못해서 인간인 우리가 학습시킬때 조절해줘야 한다
여러개중 하나가 Learning Rate다
Learning Rate가 높으면 너무 올라가버리고
내려가는데 너무 빠르거나 굴곡지면 최저값을 정하기가 어렵다
그래서 인간이 설정을 해준다

Batch Size
는 MNIST라는 데이터가 있는데 6만장의 데이터는 너무 크다
모델에도 한번에 다 데이터를 넣는게 아니라
조금씩 조금씩 나눠서 넣어주는데 몇장을 넣어줄 것인지 정해주는 것이다

Epoch/Step
앞에서 데이터를 받으면 처음부터 6만장을 다 거쳐서 봐도 그것이 끝이 아니고
여러번 반복해서 봐야된다
인공지능도 Epoch 수만큼 반복해서 봐야된다 몇만번, 몇십만번 설정해서
보게 할 수도 있다

Train/Validation/Test
전체 데이터를 받으면
Train set과 test set으로 나눈다
예로 공부하면 수능잘보려고 고등학교때 공부를 한다
수능의 특징은 test set으로 문제가 응용된 것을 평가하는것이다
test set데이터를 Train Set 앞에 있으면 안된다
그래서 데이터중 일부는 test set으로 두고 나머지는 학습용데이터로 두는 것이다
train set으로 학습하고 평가할때는 test set으로 주는 것이 좋다

Label / Ground Truth
데이터에는 정답이 있어야 한다 정답을 알려주는 것이다
모든 데이터나 레이어에 정답이 있어야되서 답지라고 할 수 있다
